Critical rules (must follow):
1) Before AOI selection, print boundary candidate names/counts with real fields, then do exact match and controlled fallback in the same dataset.
2) AOI-first hard gate: if diagnostics show `final_count==0` or empty geometry, output `WARN_APPROX_AOI` and continue the workflow in degraded mode with uncertainty annotation.
3) Compare clearly defined time windows such as before vs after.
4) Use the correct radiance band and report the unit if available.
5) Return summary differences, not only image layers.
6) For low-latency mode, keep to two short windows and output only `period_a_mean`, `period_b_mean`, `delta`.
7) Do not use `bestEffort=True`; keep scale aligned with VIIRS practice (about 500 m) and raise `maxPixels` when needed.
8) Before fixing endDate, inspect collection availability and cap to the latest complete period available in the dataset.
9) Cross-language safe logic is mandatory: never use Python native boolean operators (`not`, `and`, `or`, `!=`) on deferred `ee.*` objects.
10) In Python outputs, never call `.Not()` on `ee.ComputedObject`; use `ee.Algorithms.If` and server-side comparisons.
11) For both windows, compute `image_count` first and gate reductions by server-side conditions; if any window has no data, keep metrics null and stop delta derivation.
12) Boundary asset lock: when GAUL is used, the path must be exactly `FAO/GAUL/2015/level2`; do not rewrite as unrelated `projects/...` aliases.
    When Hong Kong district boundary is confirmed, keep `projects/ee-hku-geog7310/assets/Hong_Kong_District_Boundary` exactly; do not substitute hallucinated paths.
13) Never filter Wan Chai with only `ee.Filter.eq('NAME_EN', 'Wan Chai')` without preview; dataset values may be `Wan Chai District` or Chinese `灣仔區` only.
14) For this summary task, keep execution non-interactive: do not call `Map.addLayer`, do not paint vectors, and do not include visualization-only branches.
15) If the user asks specific years (e.g., 2020 vs 2022), use full-year windows:
    - period A: `YYYY-01-01` to `(YYYY+1)-01-01` (end-exclusive)
    - period B: same rule for the comparison year
    Do not silently switch to seasonal windows unless the user explicitly asks.

Latency-first options:
- Scope: two short windows (before/after) first
- Output shape: period_a_mean, period_b_mean, delta
- Degrade path: full window stats -> mean delta only
- Note: do NOT remove AOI by default; large/global region is usually slower.

Minimal runnable path:
- Inspect and print boundary candidate names/counts, then resolve AOI.
- Filter VIIRS monthly composites for two time windows.
- Reduce each window to a representative image.
- Compute AOI mean brightness for each period and the difference.
- Keep all branch logic server-side compatible for both JavaScript and Python backends.

Cross-language robustness (generalization guardrails):
- JavaScript-style boolean chains should not be copied directly into Python for deferred `ee` objects.
- Python generators should follow: compute window counts/flags server-side -> branch with `ee.Algorithms.If` -> avoid native Python boolean evaluation.
- If backend reports `AttributeError: 'ComputedObject' object has no attribute 'Not'`, treat as a hard rule violation and regenerate with explicit server-side branching.

Reference JavaScript snippet (GEE Code Editor):
```javascript
var aoi = ee.Geometry.Rectangle([113.95, 22.28, 114.05, 22.38]); // replace if available
var beforeStart = '2020-01-01';
var beforeEnd = '2021-01-01';
var afterStart = '2022-01-01';
var afterEnd = '2023-01-01';

var viirs = ee.ImageCollection('NOAA/VIIRS/DNB/MONTHLY_V1/VCMSLCFG')
  .filterBounds(aoi)
  .select('avg_rad');
print('band_names', viirs.first().bandNames());
print('selected_band', 'avg_rad');

var beforeImg = viirs.filterDate(beforeStart, beforeEnd).mean().rename('avg_rad');
var afterImg = viirs.filterDate(afterStart, afterEnd).mean().rename('avg_rad');

var beforeMean = beforeImg.reduceRegion({
  reducer: ee.Reducer.mean(),
  geometry: aoi,
  scale: 500,
  maxPixels: 1e13
}).get('avg_rad');

var afterMean = afterImg.reduceRegion({
  reducer: ee.Reducer.mean(),
  geometry: aoi,
  scale: 500,
  maxPixels: 1e13
}).get('avg_rad');

var delta = ee.Number(afterMean).subtract(ee.Number(beforeMean));

print('period_a_mean', beforeMean);
print('period_b_mean', afterMean);
print('delta', delta);
```

```python
boundary_asset = 'projects/ee-hku-geog7310/assets/Hong_Kong_District_Boundary'
districts = ee.FeatureCollection(boundary_asset)
print('boundary_asset', boundary_asset)
print('candidate_count', districts.size().getInfo())
print('name_en_preview', districts.aggregate_array('NAME_EN').getInfo())
print('name_tc_preview', districts.aggregate_array('NAME_TC').getInfo())

wan_chai_en = ee.List(['Wan Chai', 'Wan Chai District', 'WanChai'])
wan_chai_tc = ee.List(['灣仔', '灣仔區', '湾仔', '湾仔区'])
wan_chai = districts.filter(
    ee.Filter.Or(
        ee.Filter.inList('NAME_EN', wan_chai_en),
        ee.Filter.inList('NAME_TC', wan_chai_tc)
    )
)
print('final_count', wan_chai.size().getInfo())
if wan_chai.size().getInfo() == 0:
    print('WARN_APPROX_AOI')
else:
    aoi = wan_chai.geometry()
    # ... then VIIRS two-window mean + reduceRegion on aoi, scale=500, no bestEffort
```

Expected output:
- Period A mean, Period B mean, and difference summary
- Required diagnostics: `aoi_source`, `boundary_asset`, `candidate_count`, `final_count`, `image_count_before`, `image_count_after`, `metric_source`

Failure modes:
- Using inconsistent time windows.
- Filtering Wan Chai with only `NAME_EN == 'Wan Chai'` when the dataset stores `Wan Chai District` or Chinese-only `NAME_TC`, causing `final_count==0` and false `WARN_APPROX_AOI`.
- Not checking which radiance band should be summarized.
- Calling `.first().geometry()` on an empty filtered boundary result.
- Continuing reductions after AOI preflight failure.
- Silently replacing user-requested district/region without explicit confirmation.
- Coercing missing stats to 0 and reporting fake delta values.
- Python boolean misuse on deferred objects (`.Not()` or native `not/and/or`) can break execution with `ComputedObject` errors.
- Boundary inspect/load failure ignored (no `BLOCKED_MISSING_BOUNDARY_ASSET`) can cascade into null-feature geometry errors.
