Reference JavaScript snippet (GEE Code Editor):
```javascript
var aoi = ee.Geometry.Rectangle([113.95, 22.28, 114.05, 22.38]); // replace if available
var gsw = ee.Image('JRC/GSW1_4/GlobalSurfaceWater');
var occurrence = gsw.select('occurrence');

var threshold = 75; // persistent water threshold (%)
var persistent = occurrence.gte(threshold);

var areaM2 = persistent.selfMask().multiply(ee.Image.pixelArea()).reduceRegion({
  reducer: ee.Reducer.sum(),
  geometry: aoi,
  scale: 30,
  maxPixels: 1e13
}).get('occurrence');

print('threshold_percent', threshold);
print('persistent_water_area_km2', ee.Number(areaM2).divide(1e6));
```

Anti-patterns to avoid:

- Confusing occurrence with seasonality or transitions bands.
- Switching AOI data source silently between turns.
- Returning 0 km2 due to AOI load failure without explicit failure notice.

Expected behavior for this pattern:

- persistent_water_area_km2 remains close to the baseline (~62.47 km2), allowing small numeric variation (for example, within +/-1%).
- In sensitivity mode, outputs a threshold->area table and a short interpretation (stable vs sensitive).

Failure modes:

- Using `bestEffort=True` to hide computation issues instead of fixing AOI/data checks.
