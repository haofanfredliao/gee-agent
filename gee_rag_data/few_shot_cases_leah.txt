## FEW_SHOT: Landsat 8 C2 L2 Land Surface Temperature (Celsius)

Intent: execution
Task type: temperature inversion
Keywords: LANDSAT/LC08/C02/T1_L2, ST_B10, LST, Celsius, Kelvin, scale factor, offset, reduceRegion

User task pattern:
"Use LANDSAT/LC08/C02/T1_L2 to retrieve an image and compute land surface temperature (LST) in Celsius."

Assets in this case:
- Image collection: LANDSAT/LC08/C02/T1_L2
- Project ID: ee-hku-geog7310

Critical rules (must follow):
1) Always apply Landsat 8 C2 L2 thermal scale conversion before using ST_B10.
   - Kelvin = ST_B10 * 0.00341802 + 149.0
   - Celsius = Kelvin - 273.15
2) Do not treat raw ST_B10 DN as physical temperature.
   - ST_B10 in this dataset is scaled integer, not direct Kelvin/Celsius.
3) Use an L2 surface-temperature scene.
   - Prefer filtering by PROCESSING_LEVEL = "L2SP" before selecting the image.
4) Keep computation server-side and only materialize final small outputs.
   - Build image math first, then call getInfo() on final scalar results.

Positive ReAct trajectory:
- Observe:
  - inspect collection id and thermal band name ST_B10
  - inspect whether selected image is L2SP and note nominal scale
- Think:
  - define explicit conversion path DN -> Kelvin -> Celsius
  - ensure outputs are from converted band instead of raw ST_B10
- Act:
  - filter collection by AOI/date and select one image
  - convert ST_B10 with scale factor and offset
  - reduce region for mean LST and print Celsius result

Reference Python snippet (GEE API):
```python
import ee

ee.Initialize(project="ee-hku-geog7310")

aoi = ee.Geometry.Point([114.1694, 22.3193]).buffer(2000)

image = (
    ee.ImageCollection("LANDSAT/LC08/C02/T1_L2")
    .filterBounds(aoi)
    .filterDate("2021-01-01", "2021-12-31")
    .filter(ee.Filter.eq("PROCESSING_LEVEL", "L2SP"))
    .sort("CLOUD_COVER")
    .first()
)

st_scale = ee.Number(0.00341802)
st_offset = ee.Number(149.0)

# DN -> Kelvin -> Celsius
lst_c = (
    image.select("ST_B10")
    .multiply(st_scale)
    .add(st_offset)
    .subtract(273.15)
    .rename("LST_C")
)

mean_c = lst_c.reduceRegion(
    reducer=ee.Reducer.mean(),
    geometry=aoi,
    scale=30,
    bestEffort=True,
    maxPixels=1_000_000,
).get("LST_C")

print("Mean LST (C):", ee.Number(mean_c).getInfo())
```

Anti-patterns to avoid:
- Using ST_B10 directly as Celsius without scale/offset conversion.
- Applying only one step (e.g., multiply scale factor but forgetting offset).
- Converting to Kelvin but forgetting Kelvin-to-Celsius subtraction (273.15).
- Running reduceRegion on raw ST_B10 and reporting it as temperature.

Expected behavior for this pattern:
- The final reported LST is in Celsius and within a physically plausible range for the selected AOI/date.

## FEW_SHOT: CHIRPS Daily to Annual Precipitation Time-Series (Case 2)

Intent: execution
Task type: climate time-series aggregation
Keywords: UCSB-CHG/CHIRPS/DAILY, precipitation, time-series, annual sum, reduceRegion, mean precipitation, AOI

User task pattern:
"Use CHIRPS daily precipitation data to compute annual total precipitation for 2022 and report the mean annual precipitation over an AOI."

Assets in this case:
- Image collection: UCSB-CHG/CHIRPS/DAILY
- Project ID: ee-hku-geog7310

Critical rules (must follow):
1) Use correct time filtering for the target year.
   - For year 2022, use start="2022-01-01" and end="2023-01-01" (end-exclusive).
2) Aggregate daily precipitation with .sum() to build annual total precipitation.
   - Do not use .mean() when the task asks for annual total.
3) Use reduceRegion on the annual precipitation image to get AOI summary value.
   - The output should be a scalar precipitation value in millimeters.
4) Respect Earth Engine lazy execution.
   - Keep operations server-side and only call getInfo() for final outputs.

Positive ReAct trajectory:
- Observe:
  - inspect collection id and confirm daily precipitation band availability
  - inspect image count after date/AOI filtering
- Think:
  - map task requirement to daily->annual aggregation via .sum()
  - separate two objectives: annual total image and AOI mean scalar
- Act:
  - filter CHIRPS collection by AOI and year 2022
  - compute annual precipitation image via .sum()
  - compute AOI mean annual precipitation using reduceRegion and print final value

Reference Python snippet (GEE API):
```python
import ee

ee.Initialize(project="ee-hku-geog7310")

aoi = ee.Geometry.Point([114.1694, 22.3193]).buffer(2000)

chirps_2022 = (
    ee.ImageCollection("UCSB-CHG/CHIRPS/DAILY")
    .filterBounds(aoi)
    .filterDate("2022-01-01", "2023-01-01")
)

annual_precip = chirps_2022.sum().rename("annual_precip_mm")

mean_annual_precip = annual_precip.reduceRegion(
    reducer=ee.Reducer.mean(),
    geometry=aoi,
    scale=5566,
    bestEffort=True,
    maxPixels=1_000_000,
).get("annual_precip_mm")

print("Mean annual precipitation (mm):", ee.Number(mean_annual_precip).getInfo())
```

Anti-patterns to avoid:
- Using .mean() over the daily collection when the target output is annual total precipitation.
- Filtering year range incorrectly (e.g., using end date "2022-12-31" and missing boundary handling).
- Calling reduceRegion on a single daily image instead of the annual summed image.
- Materializing intermediate large objects with unnecessary getInfo() calls.

Expected behavior for this pattern:
- The printed precipitation value is an AOI-level annual mean (mm) derived from summed 2022 daily CHIRPS data.
