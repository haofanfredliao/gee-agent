Minimal runnable path:

- Derive masks for each encoded loss year in a short recent window first.
- Summarize annual forest loss area and return a compact year table.
- Expand to longer period only when explicitly requested by user.
- Keep all branch logic server-side compatible for both JavaScript and Python backends.

Cross-language robustness (generalization guardrails):

Reference JavaScript snippet (GEE Code Editor):
```javascript
var aoi = ee.Geometry.Rectangle([113.95, 22.28, 114.05, 22.38]); // replace if available
var gfc = ee.Image('UMD/hansen/global_forest_change_2022_v1_10');
var lossYear = gfc.select('lossyear');

// Fast default: use recent short window first.
var startYear = 2020;
var endYear = 2022;
var years = ee.List.sequence(startYear, endYear);

var yearlyLoss = ee.FeatureCollection(years.map(function(y) {
  y = ee.Number(y).toInt();
  var encoded = y.subtract(2000); // Hansen lossyear encoding
  var lossMask = lossYear.eq(encoded);

  var areaM2 = lossMask.selfMask().multiply(ee.Image.pixelArea()).reduceRegion({
    reducer: ee.Reducer.sum(),
    geometry: aoi,
    scale: 30,
    maxPixels: 1e13
  }).get('lossyear');

  return ee.Feature(null, {
    year: y,
    forest_loss_area_km2: ee.Number(areaM2).divide(1e6)
  });
}));

print('hansen_yearly_forest_loss', yearlyLoss);
```

Expected output:
- Year -> forest loss area table (fast default: recent 3 years first)
- If asked for 2023-2025, return explicit data-availability notice (no fabricated yearly values).

Failure modes:

- Promoting the entry without re-checking the exact dataset version.
- Using a long default year span (e.g., full archive) in first pass, causing slow/unstable execution.
- Returning pseudo-values for 2023-2025 instead of reporting that this Hansen profile is capped at 2022.
- Python boolean misuse on deferred objects (`.Not()` or native `not/and/or`) can break execution with `ComputedObject` errors.
- Rewriting GAUL path to `projects/...` aliases can break boundary inspect/load and cascade into null-geometry failures.
