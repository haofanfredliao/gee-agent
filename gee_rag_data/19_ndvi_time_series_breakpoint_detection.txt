Agent execution hardening (to prevent common backend failures):
- Failure signature A: `Element.geometry: Parameter 'feature' is required and may not be null.`
  - Root cause: AOI feature filter returns empty, then code calls `.first().geometry()`.
  - Required fix: compute `final_count`; only materialize geometry when `final_count > 0`; otherwise print `WARN_APPROX_AOI` and continue in degraded mode with controlled approximate AOI.
- Failure signature B: `Image.multiply: ... Got 0 and 1 bands`
  - Root cause: yearly subset is empty and `.mean()`/`.median()` creates an empty image before scaling/math ops.
  - Required fix: for each year, calculate `year_image_count` first; run reducers only when `year_image_count > 0`; keep null for empty years.
- Failure signature C: `AttributeError: 'ComputedObject' object has no attribute 'Not'`
  - Root cause: Python code applies `.Not()`/native boolean operations on deferred `ee` objects.
  - Required fix: replace with explicit server-side branching via `ee.Algorithms.If` and `ee.Number(...).gt/lt/eq`.
- Failure signature D: false `WARN_APPROX_AOI` after successful boundary inspect.
  - Root cause: client-side null check like `aoi_feature.getInfo() is None` or single-name hard match only.
  - Required fix: use alias-safe filters on `NAME_EN`/`NAME_TC`, compute `final_count` first, and prefer exact AOI geometry; when unresolved, degrade with controlled approximate AOI plus uncertainty tag.
- Output contract for validation prompts:
  1) AOI diagnostics: `aoi_source`, `boundary_asset`, `candidate_count`, `final_count`, `image_count`
  2) Method assumptions: temporal unit, scale factor, breakpoint criterion
  3) Tables: `yearly_ndvi_series`, `breakpoint_candidates`, `dominant_breakpoint`
  4) Conclusion: dominant breakpoint year and `delta_ndvi`
  5) Risks: two concise false-positive/false-negative risks

Reference JavaScript snippet (GEE Code Editor):
```javascript
var boundary = ee.FeatureCollection('projects/ee-hku-geog7310/assets/Hong_Kong_District_Boundary');
var targetAliasesEn = ee.List(['Central and Western', 'Central & Western', 'Central and Western District']);
var targetAliasesTc = ee.List(['中西區', '中西区']);
var exactEn = boundary.filter(ee.Filter.inList('NAME_EN', targetAliasesEn));
var exactTc = boundary.filter(ee.Filter.inList('NAME_TC', targetAliasesTc));
var exactMatch = exactEn.merge(exactTc);
var containsEn = boundary.filter(ee.Filter.or(
  ee.Filter.stringContains('NAME_EN', 'Central and Western'),
  ee.Filter.stringContains('NAME_EN', 'Central')
));
var containsTc = boundary.filter(ee.Filter.stringContains('NAME_TC', '中西'));
var finalMatch = ee.FeatureCollection(ee.Algorithms.If(
  exactMatch.size().gt(0), exactMatch, containsEn.merge(containsTc)
));
var finalCount = finalMatch.size();
print('final_count', finalCount);
var aoi = ee.Feature(finalMatch.first()).geometry();
var startYear = 2018;
var endYear = 2022;

var modis = ee.ImageCollection('MODIS/061/MOD13Q1')
  .filterBounds(aoi)
  .select('NDVI');

var years = ee.List.sequence(startYear, endYear);
var yearSeriesRaw = ee.FeatureCollection(years.map(function(y) {
  y = ee.Number(y).toInt();
  var start = ee.Date.fromYMD(y, 1, 1);
  var end = start.advance(1, 'year');
  var annual = modis.filterDate(start, end);
  var yearImageCount = annual.size();
  var meanVal = ee.Algorithms.If(
    yearImageCount.gt(0),
    annual.mean().multiply(0.0001).rename('NDVI').reduceRegion({
      reducer: ee.Reducer.mean(),
      geometry: aoi,
      scale: 250,
      maxPixels: 1e13
    }).get('NDVI'),
    null
  );

  return ee.Feature(null, {
    year: y,
    year_image_count: yearImageCount,
    mean_ndvi: meanVal
  });
}));

var yearSeries = yearSeriesRaw
  .filter(ee.Filter.notNull(['mean_ndvi']))
  .sort('year');

var yearList = yearSeries.toList(yearSeries.size());
var deltas = ee.FeatureCollection(ee.List.sequence(1, yearSeries.size().subtract(1)).map(function(i) {
  i = ee.Number(i).toInt();
  var prevF = ee.Feature(yearList.get(i.subtract(1)));
  var currF = ee.Feature(yearList.get(i));
  var prevMean = ee.Number(prevF.get('mean_ndvi'));
  var currMean = ee.Number(currF.get('mean_ndvi'));
  var delta = currMean.subtract(prevMean);
  return ee.Feature(null, {
    breakpoint_year: currF.get('year'),
    pre_mean_ndvi: prevMean,
    post_mean_ndvi: currMean,
    delta_ndvi: delta,
    abs_delta: delta.abs()
  });
}));

var dominant = deltas.sort('abs_delta', false).first();

print('yearly_ndvi_series', yearSeriesRaw);
print('breakpoint_candidates', deltas);
print('dominant_breakpoint', dominant);
```

Expected output:
- Breakpoint summary with fields: `breakpoint_year`, `pre_mean_ndvi`, `post_mean_ndvi`, `delta_ndvi`

Failure modes:
- No scaling for MODIS NDVI causes threshold mismatch and false breakpoints.
- Time unit inconsistency (mixing 16-day and yearly values) invalidates method assumptions.
- Null AOI feature passed to `.geometry()` causes hard failure before reduction.
- Empty annual subset can produce empty-band images and break arithmetic operations.
- Python boolean misuse on deferred objects (`.Not()` or native `not/and/or`) can break execution with `ComputedObject` errors.
- Rewriting GAUL path to `projects/...` aliases can break boundary inspect/load and cascade into null-geometry failures.
- Using `getInfo()` for AOI null checks can cause false missing-AOI branches even when aliases exist in boundary fields.
